# 特殊感染者档案（草稿）

> 说明：本文档记录当前已设计或实现的特殊感染者参数、技能与演出需求，供策划、程序、美术同步。下文示例以 Hunter 为模板，后续可按同样结构补齐其他特感。

## Hunter

> **平台适配说明**：Hunter 的所有交互设计（包括操控布局、方向指示线、能量震荡波特效、震动反馈等）均专门针对**横屏手机端（Landscape）**优化，屏幕比例为 16:9 或 18:9。不适用于竖屏模式或桌面端。横屏模式下的视野优势、按钮布局和操作习惯均已纳入设计考量。

- **定位**：高机动突袭 / 压制单体  
- **角色设计详述**：  
  Hunter 是团队中的尖兵与刺客，凭借无与伦比的机动性在复杂地形中穿梭。其核心战术在于寻找落单或站位不佳的幸存者，通过精准的飞扑瞬间瓦解对方的阵型，制造以多打少的机会。一个优秀的 Hunter 能持续对幸存者施加心理压力，迫使他们在移动中不断分心戒备。

- **基础属性**

  | 参数 | 数值 | 设计说明 |
  | ---- | ---- | -------- |
  | 生命值 | 250 | 非常脆弱，强调 “一击即走” 的刺客哲学，无法承受正面火力。 |
  | 移动速度 | 300 | 基础移速较高，便于快速调整位置，接近目标。 |
  | 冲刺速度 | 700 | 极快的冲刺速度，是核心机动性的体现，用于快速接近或撤离。 |
  | 护甲 | 0 | 没有任何减伤，进一步强化脆皮特性。 |

- **技能：高速飞扑（pounce）**
  1. **技能描述**  
     Hunter 蹲伏蓄力，随后如同一支离弦之箭般向目标方向高速飞扑。一旦命中幸存者，将立即将其扑倒并施加压制，是 Hunter 最具威胁的单体控制技能。

  2. **核心机制**
     - 冷却时间：6 秒  
       - 设计思路：较短冷却允许频繁尝试，但一旦扑空，猎手会在 6 秒内陷入脆弱期，风险与收益并存。
     - **蓄力机制：决定飞扑距离**（横屏手机端专用）
       - **蓄力方式**：玩家长按【蓄力飞扑】按钮（位于横屏右侧下方），Hunter 进入蓄力状态，移动速度降低。
       - **蓄力时长**：0-0.8 秒，蓄力时间越长，飞扑距离越远。
       - **方向指示线**（横屏手机端特有 UI）：
         - 蓄力期间，Hunter 操控者屏幕会显示一根方向指示线，从 Hunter 位置延伸至目标方向（跟随左侧虚拟摇杆方向）。
         - 指示线长度根据蓄力时长动态变化，最大长度不超过屏幕宽度的 60%，确保在横屏视野内完整显示。
         - 指示线颜色随蓄力进度变化：暗红（初期）→ 鲜红（中期）→ 亮橙（饱和期），与震荡波颜色同步。
         - 指示线末端显示预估落点标记，帮助玩家预判飞扑距离。
       - **幸存者预警**：人类可以通过观察 Hunter 的朝向（头部方向）判断飞扑方向，蓄力时 Hunter 的动画特效为周边类似赛亚人的冲击波特效（红色冲击波），冲击波转得越快代表飞扑距离越远。
     - **施法前摇：蹲伏蓄力（动态）**
       - 动作特效：身体下蹲，核心能量源（胸口或背部）开始脉动，周身持续扩散猩红色球面波纹。
       - **能量震荡波视觉表现**：
         - **初期（0-0.3 秒）**：波纹扩散速度慢、间距大、颜色较暗。
         - **中期（0.3-0.7 秒）**：波纹扩散速度急剧加快、间距紧密、颜色变为鲜红，仿佛能量即将失控。
         - **饱和期（0.7-0.8 秒）**：波纹频率达到极限，几乎重叠在一起，颜色变为刺眼的亮红色/橙色，并且每次震荡都会迸发出细小的闪电粒子。这是即将飞扑的终极信号。
       - **释放瞬间**：松开按钮，Hunter 立刻向当前左摇杆方向飞扑出去。原本向四周扩散的震荡波会在飞扑瞬间全部收束到 Hunter 身后，形成一道凝聚的红色尾迹。
       - 音效描述：  
         - 起始：低沉威胁的喘息与肌肉绷紧声。  
         - 蓄力：能量聚集的嗡鸣声，音调逐渐升高，与波纹频率同步。  
         - 可释放提示：蓄满时有短促 "咔嚓" 或 "啵" 声，提醒玩家可以施放。  
       - 设计思路：明显前摇与音效为幸存者提供反应窗口，考验 Hunter 对时机与对方硬直的把握。通过观察震荡波的颜色、频率和速度，幸存者可以准确判断此次飞扑的蓄力程度和威胁大小。

  3. **命中效果**
     - **初始伤害**：25 点（重在人群控制，伤害用于破甲或补刀）。  
     - **冲击波效果**：飞扑命中后，被飞扑中的单位周边小范围内（约 3 米）的其他单位会进入"跌跌撞撞"状态，持续 1 秒。此期间移动速度降低、视角抖动，无法正常瞄准，强调冲击力与团控威胁。
     - **压制持续**：直至 Hunter 或被压制者死亡，或 Hunter 长按【蓄力飞扑】按钮主动解除（横屏手机端）。  
     - **压制机制**：  
       - 被压制者 HUD 变为灰红色调，边缘呈现脉搏特效，中央提示 “你被 Hunter 压制了！挣扎或等待救援！”  
       - 队友视角中，被扑倒者会显示醒目的红色标记与高亮轮廓。  
       - **救援流程**（横屏手机端）：靠近被压制队友后，屏幕中央会出现【救援】按钮，长按 1.5 秒完成救援（过程中若被打断则重置进度）；救援成功 Hunter 短暂硬直。  
       - **自救挣扎**（横屏手机端）：被扑者屏幕会显示【挣扎】按钮，快速连击该按钮可尝试提前挣脱，耗时约 3 秒、成功率低，主要用于牵制 Hunter 或为队友创造输出窗口。  
       - 设计思路：强调团队协作；幸存者需要在“救援”与“集火”之间做取舍。

  4. **视觉与动画要素**
     - 飞扑轨迹：Hunter 化为模糊残影，拖曳猩红/亮黄能量尾迹，持续约 0.5 秒。  
     - 命中特效：撞击点爆出小范围红色冲击波粒子；Hunter 利爪扣住幸存者，能量沿身体传导。  
     - 音效：  
       - 碰撞：沉重肉体撞击 + 布料撕裂。  
       - 幸存者：短促痛呼或被扼住喉咙的窒息声。  
       - Hunter：得意、残忍的低吼/嘶鸣。  
     - 压制动画：  
       - 被击倒瞬间，幸存者身体后仰、双脚离地。  
       - 压制期间，幸存者仰躺不停挣扎，Hunter 俯身按住并低吼，强调压迫感。

  5. **横屏手机端操控与特效方案（能量震荡波版）**
     > **平台说明**：以下所有交互设计均专门针对横屏手机端（Landscape）优化，屏幕比例为 16:9 或 18:9，不适用于竖屏或桌面端。
     
     - **横屏手机端核心操控布局**
       - **屏幕左侧（约 1/4 宽度）**：虚拟摇杆，控制移动和面向。摇杆位置固定在左下角区域，避免与游戏画面中心冲突。
       - **屏幕右侧（约 1/4 宽度）**：两个按钮垂直排列，上方为普攻按钮，下方为蓄力飞扑按钮。按钮大小需适配拇指操作，建议直径 60-80px，间距 20px。
       - **屏幕中央（约 1/2 宽度）**：游戏画面区域，确保 Hunter 和方向指示线清晰可见。
       - **横屏优势**：横向视野更广，便于观察 Hunter 的朝向和震荡波扩散，方向指示线可充分利用横向空间延伸。
     - **蓄力飞扑与能量震荡波特效**
       - **蓄力启动**：玩家长按【蓄力飞扑】按钮，Hunter 进入蓄力状态，移动速度降低。
       - **能量震荡波视觉表现**：
         - Hunter 身体下蹲，核心能量源（胸口或背部）开始脉动。
         - 随着蓄力，一道道半透明的猩红色球面波纹，以 Hunter 为中心，持续而快速地向四周扩散震开，如同石子投入水面形成的涟漪，但充满力量感。
         - **动态变化**：
           - **初期（0-0.3 秒）**：波纹扩散速度慢、间距大、颜色较暗。
           - **中期（0.3-0.7 秒）**：波纹扩散速度急剧加快、间距紧密、颜色变为鲜红，仿佛能量即将失控。
           - **饱和期（0.7-0.8 秒）**：波纹频率达到极限，几乎重叠在一起，颜色变为刺眼的亮红色/橙色，并且每次震荡都会迸发出细小的闪电粒子。这是即将飞扑的终极信号。
       - **释放与方向**：松开按钮，Hunter 立刻向当前左摇杆方向飞扑出去。原本向四周扩散的震荡波会在飞扑瞬间全部收束到 Hunter 身后，形成一道凝聚的红色尾迹。
     - **幸存者视角的预警**
       - 幸存者可以清晰地看到 Hunter 周身不断加速扩散的红色震荡波。
       - 通过观察震荡波的颜色、频率和速度，可以准确判断此次飞扑的蓄力程度和威胁大小。波纹越快越亮，代表飞扑距离越远，需要立刻寻找掩体或准备反击。
     - **横屏手机端优化补充**
       - **锁定辅助**：蓄力时，摇杆方向若大致对准幸存者（角度偏差 < 30°），提供轻微方向吸附，降低横屏操作精度要求。
       - **震动反馈**（横屏手机端特有）：
         - 每扩散出一圈重要的震荡波（如每 0.2 秒），提供一次轻微的同步脉冲震动（10-20ms），让玩家"感觉"到能量的涌动。
         - 蓄力达到最大：一次强烈的长震动（100-150ms），提示可以释放。
         - 成功命中：强烈的打击震动（200ms），增强打击感。
       - **横屏 UI 布局**：
         - HUD 元素（血量、技能冷却）位于屏幕上下边缘，不遮挡中央游戏区域。
         - 方向指示线和震荡波特效优先显示，确保在横屏视野中清晰可见。
       - **性能优化**：
         - 震荡波粒子数量根据设备性能动态调整（低端设备减少波纹数量，保持流畅度）。
         - 横屏模式下，视野范围更大，需优化远距离对象的渲染性能。

  6. **横屏手机端程序/美术协作要点**（建议项）
     - **横屏适配核心要求**：
       - 所有 UI 元素和交互控件需针对横屏比例（16:9/18:9）进行布局优化。
       - 虚拟摇杆和按钮位置需适配横屏拇指操作区域，避免误触。
       - 方向指示线需考虑横屏视野范围，确保完整显示且不超出屏幕边界。
     - **性能优化**：
       - 粒子与尾迹需池化，避免性能抖动。横屏模式下视野更广，需控制同时存在的粒子数量。
       - 能量震荡波特效需要逐帧计算扩散速度与颜色变化，确保视觉反馈与蓄力时长精确对应。低端设备需动态降级波纹数量。
     - **交互实现**：
       - HUD 提示、救援交互、挣扎逻辑需与 UI/音效协同实现，横屏布局需确保不遮挡关键信息。
       - 方向指示线需实时更新，跟随摇杆方向变化，长度需根据蓄力时长和屏幕宽度动态计算。
       - 震动反馈需与波纹扩散频率同步，提供触觉反馈。需检测设备是否支持震动 API。
     - **横屏特有优化**：
       - 按钮大小和间距需适配不同尺寸的横屏设备（5-7 英寸屏幕）。
       - 方向指示线和震荡波特效需在不同横屏分辨率下保持视觉一致性。
       - 考虑横屏模式下双手握持设备的操作习惯，优化按钮布局。

### 高危状态与成长机制设计

#### 维度一：状态型伤害加成

1. **落单惩戒**
   - 触发条件：飞扑命中后，10 米内无其他存活队友。  
   - 视觉 / 听觉：  
     - Hunter 端：幽蓝屏幕边缘闪烁 + 回音狼嚎。  
     - 被压制者：环境变黑暗，仅有风声与剧烈心跳。  
   - 效果：对目标造成的全部伤害 +50%，一旦队友进入范围立即解除。

2. **协同猎杀**
   - 触发条件：另一特感（如 Smoker、Jockey）正控制或攻击该幸存者或其近邻队友。  
   - 视觉 / 听觉：  
     - Hunter 端：屏幕现暗紫光环，伴随同步心跳音效。  
   - 效果：Hunter 爪击伤害 +25%，鼓励团队协同。

#### 维度二：成长型伤害 —— 「血腥撕裂」流血 Debuff

- **触发**：每次成功爪击叠一层。  
- **视觉**：目标血条下出现层数图标；屏幕血迹随层数加剧；伤口粒子增多。  
- **范围冲击**：飞扑命中后，被压制目标周围小范围（约 3 米）内的幸存者会被冲击波影响 1 秒，表现为跌跌撞撞、移动输入被扰动并附带轻微视角抖动，强调冲击力与团控威胁。  
- **伤害曲线（每层叠加）**：

  | 流血层数 | 每秒伤害 | DPS（假设 2.5 次/秒） | 反馈 |
  | -------- | -------- | --------------------- | ---- |
  | 1-3 层 | 2/层 | 5 → 10 → 15 | 轻微血迹、轻微呻吟 |
  | 4-6 层 | 5/层 | 30 → 45 → 60 | 血液流淌、呻吟加重 |
  | 7-9 层 | 10/层 | 90 → 120 → 150 | 大量喷溅、惨叫 |
  | 10 层+ | 15/层 | 195+ | 屏幕血红、惨叫不断 |

- **重置**：压制被解除时每秒清除 5 层，清除阶段伤害仍持续。

#### 维度三：操控者与受害者体验

- **Hunter HUD**：  
  - 实时 DPS 大字报；  
  - 「血腥撕裂」层数计数；  
  - 「落单惩戒」与「协同猎杀」图标高亮；  
  - 处决动画（压制击杀后），同时获得短时间伤害增益。

- **幸存者视角**：  
  - 「绝望量表」心跳仪随层数快速灌血；量表满立即死亡；  
  - **自救**（横屏手机端）：快速连击屏幕上的【挣扎】按钮可减缓流血层数叠加速度，但无法完全停止；  
  - **救援提示**（横屏手机端）：队友靠近时，屏幕显示"队友正在救援！坚持住！"并显示倒计时进度条，受 Hunter 攻击干扰时进度条会剧烈抖动甚至短暂回退。

#### 战术总结与平衡

- **Hunter 策略**：优先猎孤、配合队友、判断持续压制还是撤离。  
- **幸存者反制**：集中火力、善用控制道具、及时救援。  
- **整体定位**：将 Hunter 从纯控制者升级为极具成长性的终结者。

### Hunter 专属弱点设计

1. **空中易伤**
   - 机制：飞扑在空中时受到 200% 伤害。  
   - 反馈：  
     - 幸存者命中有强化火花 / 音效；  
     - Hunter 屏幕闪黄红边，伴护甲破裂声。  
   - 意图：鼓励预判射击，要求 Hunter 谨慎起跳。

2. **推击眩晕**
   - 机制：被近战推中时，立即击飞并眩晕 3 秒（10% 移速、全操作锁定）。  
   - 反馈：  
     - 幸存者听到“叮”提示；  
     - Hunter 端黑白模糊 + 耳鸣。  
   - 意图：提供高风险反制手段。

3. **墙体弹反**（横屏手机端）
   - **机制**：飞扑撞墙后 0.3~0.5 秒判定窗口内，再次按下【蓄力飞扑】按钮即可触发弹反。  
   - **成功**：刷新冷却、前摇缩短、尾迹变亮（气浪粒子 + 音爆），可立即朝当前摇杆方向发起第二次飞扑。  
   - **失败**：1.5 秒硬直，播放撞击闷哼，飞扑进入完整冷却。  
   - **视觉提示**（横屏手机端）：撞墙瞬间，屏幕中央会出现闪烁的【蓄力飞扑】按钮提示图标，并伴随清脆的"铮！"提示音。  
   - **意图**：提高操作上限与观赏性，为顶尖 Hunter 玩家提供连续高速位移的秀操作空间。
